## Section 1 : FAQ (frequently asked questions)

#### Q1: What are IT security frameworks?
IT security frameworks are structured guidelines that define policies, procedures, and processes to manage and mitigate information security risks. Examples include ISO 27001, NIST SP 800-53, and COBIT.

#### Q2: Why is it important for organizations to comply with IT security frameworks?
Compliance with IT security frameworks ensures that organizations have robust security measures in place to protect sensitive data, meet regulatory requirements, and reduce the risk of security breaches.

#### Q3: How do I choose the right IT security framework for my organization?
The choice of an IT security framework depends on factors like the industry, regulatory requirements, the size of the organization, and specific security needs. For example, healthcare organizations often use HITRUST due to its alignment with HIPAA.

## Section 2: Responses to the Questions

#### Q1: Which of the frameworks do you think would be applicable to the following organisations:
 - International bank.
 - Large hospital.
 - Large food manufacturing factory.

<br>

### International Bank

| **Framework** | **Justification** | 
| :----------- | :----------------- |
| ISO 27001 & ISO 27002 | Provides a comprehensive set of controls for establishing an information security management system (ISMS). |
| NIST SP 800-53 | Helps in managing and mitigating security risks, widely recognized for its rigorous standards. |
| COBIT | Focuses on governance and management of enterprise IT, ensuring alignment with business goals and regulatory requirements. |
| PCI DSS | Essential for protecting cardholder data and ensuring secure payment processes. |

<br> 

### Large Hospital

| **Framework** | **Justification** | 
| :----------- | :----------------- |
| ISO 27001 & ISO 27799 | ISO 27799 specifically addresses information security in healthcare. |
| NIST SP 800-53 & NIST SP 800-171 | Provides security controls for protecting patient data and complying with federal regulations. |
| HITRUST CSF | Tailored for the healthcare sector, integrating various standards like HIPAA, NIST, and ISO. |
| GDPR | Ensures the protection of EU citizens' personal data. |

<br> 

### Large Food Manufacturing Factory

| **Framework** | **Justification** | 
| :----------- | :----------------- |
| ISO 27001 | Establishes an ISMS to protect sensitive information. |
| NIST SP 800-53 | Provides a broad range of security controls applicable to various industries, including manufacturing. |
| COBIT | Ensures IT governance and management align with business goals and regulatory requirements. |
| GDPR | Ensures compliance with data protection regulations, especially if the factory handles EU citizens' data. |

#### Q2: Summarise the tests and recommendations you would make to the owners/ managers for each of the above businesses to help them use the frameworks and comply with industry standards.

<br> 

### International Bank

| **Tests** | **Detail** |
| :----------- | :----------------- |
| Vulnerability Assessments | Regularly conduct vulnerability assessments to identify and mitigate security weaknesses. |
| Penetration Testing | Perform penetration tests to evaluate the effectiveness of security controls. |
| Compliance Audits | Conduct audits to ensure compliance with frameworks like PCI DSS and ISO 27001. |

| **Recommendations** | **Detail** |
| :----------- | :----------------- |
| Implement an ISMS | Adopt ISO 27001 to establish a robust ISMS. |
| Regular Training | Conduct regular cybersecurity training for employees to maintain awareness of security policies and procedures. |
| Continuous Monitoring | Utilize continuous monitoring tools to detect and respond to security incidents promptly. |

<br> 

### Large Hospital

| **Tests** | **Detail** |
| :----------- | :----------------- |
| Risk Assessments | Perform regular risk assessments to identify and address potential threats to patient data. |
| Access Control Reviews | Regularly review access controls to ensure only authorized personnel can access sensitive information. |
| Compliance Checks | Conduct compliance checks to ensure adherence to frameworks like HIPAA and HITRUST CSF. |

| **Recommendations** | **Detail** |
| :----------- | :----------------- |
| Adopt HITRUST CSF | Implement HITRUST CSF to align with healthcare-specific security requirements. |
| Data Encryption | Ensure all patient data is encrypted both in transit and at rest. |
| Incident Response Plan | Develop and regularly update an incident response plan to manage data breaches effectively. |

<br> 

### Large Food Manufacturing Factory

| **Tests** | **Detail** |
| :----------- | :----------------- |
| Security Audits | Conduct security audits to evaluate compliance with frameworks like ISO 27001 and NIST SP 800-53. |
| Supply Chain Assessments | Assess the security of supply chain partners to ensure they comply with the factory’s security policies. |
| Network Security Testing | Regularly test the network for vulnerabilities and implement necessary patches. |

| **Recommendations** | **Detail** |
| :----------- | :----------------- |
| Implement ISO 27001 | Establish an ISMS using ISO 27001 to protect sensitive information. |
| Regular Training | Provide cybersecurity training to employees, particularly those involved in handling sensitive data. |
| Vendor Management | Ensure that all third-party vendors comply with the factory’s security standards and regularly assess their security posture. |
